The problem

A codebase you didn't write, an AI you can't fully trust.

You inherit a 200k-line project. You ask an AI to add one feature. It does — but the next deploy breaks something you never asked it to touch. Sound familiar?

1 A client hands you 200,000 lines of code you've never read.
2 You ask an AI to make a small change in one module.
3 The AI silently edits a shared utility you didn't know existed.
4 Three modules break. No one notices until staging. Or worse — until production.
The fix

Map first. Then modify — only inside the lines.

map-then-modify is a protocol your AI follows. It produces a complete architectural map of your codebase, then enforces precise boundaries every time it changes code.

step 01 · map

Build a map of the whole codebase.

A recursive analysis produces an .architecture/ wiki — stable module IDs, precise file and line boundaries, interaction edges, risks, and confidence ratings on every claim.

step 02 · plan

Declare boundaries before touching code.

Every change starts with a written dev plan. The AI lists which files and lines it will edit — and which it explicitly will not. No silent surprises.

step 03 · modify

Edits stay strictly inside the plan.

If reality diverges from the map mid-edit, the wiki self-heals first, the plan expands transparently, and the change continues — without ever breaking what isn't documented.

No more silent regressions in modules nobody told the AI to change.
The map keeps up with the code on every successful change.
Risks, gotchas, and dead code surface early — never quietly.
You stay in the loop — the AI tells you what it healed and why.
Install

One command. Any agent platform.

Clone the repo into your agent's skills directory. Restart your agent. That's it.

# pick the skills directory for your agent (see table below)
git clone git@github.com:Luis-yl/map-then-modify.git \
    ~/.claude/skills/map-then-modify

Don't use Claude Code? Replace ~/.claude/skills/ with whichever skills directory your agent uses.

Platform
Skills directory
Claude Code
~/.claude/skills/
Codex CLI
~/.codex/skills/
Cursor / VS Code with Copilot
.cursor/skills/ or .copilot/skills/
Gemini CLI
~/.gemini/skills/
OpenCode / Cline / others
follow each platform's skills-loader convention

Once installed, just tell your agent: "map this project" — or any wording like "啃下这个项目", "safe refactor in an unfamiliar repo". The skill auto-triggers.

Stop guessing. Start mapping.

It's free, open-source, and works with the agent you already use.
